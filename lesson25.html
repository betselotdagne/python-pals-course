What Happens When Code Breaks?

When something goes wrong in your Python code‚Äîlike trying to divide by zero or asking for text input when you needed a number‚ÄîPython stops running and shows a scary error message (called a **Traceback**).

We don't want our program to crash! We want to give the user a helpful message and keep going. This is called **Error Handling**, and we use the **`try`** and **`except`** keywords.

The `try` and `except` Blocks

  • The **`try`** block holds the code that **might** cause an error.
  • If an error happens in the `try` block, Python immediately skips the rest of the `try` code and jumps to the **`except`** block.
  • The **`except`** block holds the "plan B" code‚Äîwhat to do if the error happens (like printing a friendly message).

Example 1: Handling a Division by Zero


num1 = 10
num2 = 0 # This will cause a ZeroDivisionError!

try:
    # Try to do the math
    result = num1 / num2
    print(result) # This line will be skipped!
    
except:
    # If a crash happens, run this code instead!
    print("Oops! You can't divide by zero.")

Example 2: Specific Error Types

You can tell Python which specific error to catch. The common one when converting input is **`ValueError`**. 


age_text = input("Enter your age:")

try:
    age = int(age_text) # This might fail if the user types "ten"
    print("You are " + str(age) + " years old.")
    
except ValueError:
    # This only catches the ValueError, not other errors.
    print("That was not a valid number! Try again.")
