The Need for `elif`

In Lesson 7, we learned about `if` and `else`‚Äîa two-way choice (yes or no). But what if your program needs to choose between **many** different options?

This is where **`elif`** comes in. It stands for **"Else If"** and lets you check a second, third, or even tenth rule, but only if the previous rules were all **False**.

How the computer checks `if/elif/else`:

The computer reads the rules from top to bottom. As soon as it finds the **first** rule that is **True**, it executes that block of code and then **skips the rest** of the `elif` and `else` checks.

Example 1: Multiple Choices (A Grading System)


grade = 85

if grade >= 90:
    print("You got an A!")
elif grade >= 80:
    print("You got a B!")
else:
    print("You got a C or lower.")

(The computer checks: Is 85 >= 90? No. Is 85 >= 80? Yes! It prints "You got a B!" and stops checking.)

Example 2: Order Matters!

If we changed the order and put the smallest number first, it would behave differently. Always check the stricter, higher value conditions first!
